206. Reverse Linked List


/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode() : val(0), next(nullptr) {}
 *     ListNode(int x) : val(x), next(nullptr) {}
 *     ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */
class Solution {
public:
    ListNode* reverseList(ListNode* head) {
        // 1. 初始化兩個起點指標
        ListNode* curr = head;    /*
        curr 站在第一棟房子 head 是指向第一個node 而不是head的next 是第一個node 
        */ 
        ListNode* prev = nullptr; // prev 代表 curr 的背後 (空的)

        // 2. 只要 curr 還沒走到盡頭，就繼續反轉
        while (curr != nullptr) {
            // 動作 D：先派替身把未來的路記住 (這必須是第一步！)
            ListNode* next_temp = curr->next; 

            // 動作 A：把目前節點的箭頭反轉，指向背後的 prev
            curr->next = prev;

            // 動作 B：prev 往前走一步 (來到 curr 的位置)
            prev = curr;

            // 動作 C：curr 往前走一步 (利用剛才備份的救命繩)
            curr = next_temp;
        }

        // 3. 迴圈結束時，curr 已經跑到 nullptr (越界了)
        // 此時 prev 剛好停在原本的最後一個節點，也就是「新的頭節點」！
        return prev; 
    }
};

/*

我們用最嚴謹的 C++ 底層機制，來解開你的這兩個重大疑問：

### 疑問一：為什麼你以為 `head->next` 才是第一個 Node？

你會有這個印象非常正常，因為在學校教資料結構，或是在解某些比較複雜的 Linked List 題目時，我們常常會刻意創造一個 **「虛擬頭節點 (Dummy Node)」**。
那種情況下，第一個節點真的是空的，`head->next` 才是真正裝著第一筆資料的節點。

**但在 LeetCode 206 題（以及絕大多數的基礎題）中，預設是沒有 Dummy Node 的。**
LeetCode 傳給你的 `head`，就是一張**直接寫著「第一個真實節點」位址**的紙條。
*   `head` 拿著 `0x1000`。
*   位址 `0x1000` 的房子裡，就實實在在地裝著 `val = 1`。
所以，`curr = head` 才會直接讓 `curr` 拿到第一個真實節點的位址。


### 疑問二：既然 `curr` 只是位址，它怎麼會有 `next`？

這就是 C++ 語法糖（Syntactic Sugar）施展魔法的地方了。

你說得很對，位址本身沒有 `next`。真正有 `next` 的，是位址指向的那棟「房子（實體物件）」。

在 C++ 裡面，如果要透過位址去拿房子裡的東西，必須做**兩個連續的嚴謹動作**：
1.  **解引用的鑰匙 (`*`)**：拿著位址走到那裡，把房門打開。`*curr` 代表的才是「那棟房子本身」。
2.  **拿取成員的點 (`.`)**：走進房子後，拿出裡面的變數。所以要寫成 `(*curr).next`。

但是在程式碼裡到處寫括號和星號 `(*curr).next` 實在太容易眼花和打錯了。所以 C++ 的發明者創造了一個專屬的符號：**箭頭 `->`**。

**`curr->next` 其實是一句「濃縮指令」，它在底層等同於 `(*curr).next`。**

當你寫下 `curr->next = prev;` 的時候，C++ 編譯器幫你做的事情是：
1.  「看著 `curr` 手上的位址 (`0x1000`)。」
2.  「順著位址走過去，打開那棟房子的門 (`*`)。」
3.  「找到房子裡面那個叫做 `next` 的變數 (`.`)。」
4.  「把 `next` 裡面的資料，替換成 `prev` 手上的位址。」
