### 快慢指標的 C++ 寫法

我們派兩個替身指標，一開始都站在起點 (`head`)：
* **慢指標 (`slow`)**：每次走 1 步。
* **快指標 (`fast`)**：每次走 2 步。

```cpp
ListNode* slow = head;
ListNode* fast = head;

// 仔細看這個 while 條件，是不是覺得很眼熟？
while (fast != nullptr && fast->next != nullptr) {
    slow = slow->next;          // 慢指標走 1 步
    fast = fast->next->next;    // 快指標走 2 步
}
```

### 為什麼是這個 `while` 條件？（呼應你昨天的頓悟！）

昨天我們說過：「只要程式碼出現了 `->next->something`，就一定要先檢查 `->next` 存不存在」。

在這裡，`fast` 每次都要跨兩步 (`fast->next->next`)。
為了保證他不踩空（不觸發 Runtime Error），我們必須嚴謹地確認：
1.  `fast != nullptr` (確認目前這棟房子在)
2.  `fast->next != nullptr` (確認下一棟房子也在，這樣他才能安心跨到下下棟)

只要前面沒路了（上述任一條件不成立），迴圈就會立刻煞車。

### 具體停在哪裡？我們來嚴謹推演一次

這個迴圈煞車的時候，會因為串列的長度是「奇數」還是「偶數」而有微小的差異：

**情況一：長度是奇數（例如 `1 -> 2 -> 3 -> 2 -> 1`）**
* **初始：** `slow` 站在 1，`fast` 站在 1。
* **第一圈：** `slow` 走到 2，`fast` 走到 3。
* **第二圈：** `slow` 走到 3，`fast` 走到最後一個 1。
* **煞車：** 此時 `fast->next` 是 `nullptr`（迴圈結束）。
* **結果：** `slow` 非常完美地停在正中間的 `3` 身上！

**情況二：長度是偶數（例如 `1 -> 2 -> 2 -> 1`）**
* **初始：** `slow` 站在第一個 1，`fast` 站在第一個 1。
* **第一圈：** `slow` 走到第一個 2，`fast` 走到第二個 2。
* **第二圈：** `slow` 走到第二個 2，`fast` 跨出界線，變成 `nullptr`。
* **煞車：** 此時 `fast` 是 `nullptr`（迴圈結束）。
* **結果：** `slow` 剛好停在「後半段隊伍的第一個人」身上（也就是第二個 `2`）。

### 任務無縫接軌的嚴謹邏輯

想像一下：
1.  **任務一（找中點）：** `slow` 和 `fast` 跑完賽跑後，`fast` 已經走到盡頭功成身退了。而 `slow` 完美地停在下半場的起點（例如 `1 -> 2 -> 3 -> 2 -> 1`，`slow` 停在 `3`）。
2.  **任務二（反轉下半場）：** 現在，我們直接把這個 `slow` 當作下半場的 `head`。然後，我們直接在這裡**原封不動地默寫出你最熟悉的反轉迴圈**！

你看，我們把任務一跟任務二接起來，程式碼會長這樣：

```cpp
// ------ 任務一：龜兔賽跑找中點 ------
ListNode* slow = head;
ListNode* fast = head;
while (fast != nullptr && fast->next != nullptr) {
    slow = slow->next;
    fast = fast->next->next;
}
// 迴圈結束時，slow 站在下半場的起點


// ------ 任務二：反轉下半場 ------
ListNode* prev = nullptr;
ListNode* curr = slow;    // 直接把 slow 交接給 curr，當作反轉的起點！

while (curr != nullptr) {
    ListNode* next_temp = curr->next; // 派出第三個指標 next_temp 當救命繩
    curr->next = prev;
    prev = curr;
    curr = next_temp;
}
// 迴圈結束時，prev 站在整條串列的最後面，成為下半場「新的火車頭」！
```

### 兩軍對壘的畫面形成了

經過這兩個步驟，你現在手上有兩個「火車頭」了：
* **左邊的火車頭**：一開始傳進來的 `head`。它帶著前半段的隊伍往右指。
* **右邊的火車頭**：剛剛反轉完畢回傳的 `prev`。它帶著後半段的隊伍往左指。

/**
 * Definition for singly-linked list.
 * struct ListNode {
 * int val;
 * ListNode *next;
 * ListNode() : val(0), next(nullptr) {}
 * ListNode(int x) : val(x), next(nullptr) {}
 * ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */
class Solution {
public:
    bool isPalindrome(ListNode* head) {
        // ==========================================
        // Step 1（找中點）： 龜兔賽跑 (快慢指標)
        // ==========================================
        ListNode* slow = head;
        ListNode* fast = head;
        
        // 【嚴謹邊界】因為 fast 每次要走兩步 (fast->next->next)
        // 必須確保「目前這棟房子」和「下一棟房子」都存在，才不會觸發 Null Pointer 錯誤
        while (fast != nullptr && fast->next != nullptr) {
            slow = slow->next;       // 慢指標走 1 步
            fast = fast->next->next; // 快指標走 2 步
        }
        // 迴圈結束時，fast 跑到終點，slow 剛好停在正中間（下半場的起點）

        // ==========================================
        // Step 2（反轉）： 反轉後半段的鏈結串列
        // ==========================================
        ListNode* curr = slow;     // 直接把 slow 當作反轉任務的起點
        ListNode* prev = nullptr;  // prev 用來接收反轉後的箭頭，初始為 nullptr (未來的尾巴)
        
        // 【嚴謹邊界】反轉必須走到最後一個節點掉出界外 (nullptr) 為止，確保最後一棟房子也有被反轉到
        while (curr != nullptr) {
            ListNode* next_temp = curr->next; // 先派替身記住未來的路，避免斷鏈
            curr->next = prev;                // 核心動作：把箭頭切斷，指回上一個節點
            prev = curr;                      // prev 往前走一步
            curr = next_temp;                 // curr 往前走一步
        }
        // 迴圈結束時，curr 掉出界外，prev 剛好站在反轉後鏈結串列的「新火車頭」

        // ==========================================
        // Step 3（比對）： 左右兩半部對對碰
        // ==========================================
        ListNode* head1 = head; // 左半邊的火車頭 (原本的起點)
        ListNode* head2 = prev; // 右半邊的火車頭 (反轉後的起點)
        
        // 只要兩邊都還沒走到盡頭，就一格一格比對數字
        while (head1 != nullptr && head2 != nullptr) {
            if (head1->val != head2->val) {
                return false; // 只要有一個數字不一樣，就絕對不是迴文，提早結束
            }
            // 數字一樣，兩人同時牽手往下走一步
            head1 = head1->next;
            head2 = head2->next;
        }
        
        // 如果能安然無恙走完整個迴圈，代表全部數字都對稱！
        return true;
    }
};
