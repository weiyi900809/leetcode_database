
169. Majority Element (Easy)
題目目標： 給你一個陣列 nums，裡面有一個「多數元素」（定義是：出現次數嚴格大於 $\lfloor n / 2 \rfloor$ 次）。
請你把它找出來。
破關提示（摩爾投票法 Boyer-Moore Voting Algorithm）：
空間複雜度 $O(1)$ 的極致解法。
我們用嚴謹的狀態機來推演，請想像這是一個「打擂台」的過程，因為目標元素的數量超過一半，所以就算它跟其他所有不同的數字「一換一（互相抵銷）」，最後活下來的一定還是它。
你需要設定兩個狀態變數：
candidate（目前的擂台主）
count（擂台主的血量，初始為 0）
核心邏輯（for 迴圈掃描）：
如果 count == 0：代表擂台上沒人，現在這個數字直接成為新的 candidate，並且 count = 1。
如果現在的數字跟 candidate 一樣：血量加一（count++）。
如果現在的數字跟 candidate 不一樣：觸發一換一抵銷，血量減一（count--）。
最後迴圈跑完，留在 candidate 裡的值，就是絕對的多數！

這題這個解法是gemini 告訴我 我才知道可以這樣寫 真的神奇的解法


class Solution {
public:
    int majorityElement(vector<int>& nums) {
        int count=0;
        int arr_size=nums.size();
        int candidate=0;
        for(int i=0;i<arr_size;i++){
            if(count==0){
                candidate=nums[i];
                count++;
            }
            else{
                if(candidate!=nums[i]){
                    count--;
                }
                else{
                    count++;
                }
            }
        }
        return candidate;
    }
};

### 第二題：344. Reverse String (Easy) - 跨入 String 階段

* **題目目標：** 系統給你一個字串（在 C++ 裡面其實就是字元的陣列 `vector<char>`），請你在**不配置新陣列**的情況下，把字串原地左右顛倒。空間複雜度必須是 $O(1)$。
* **破關提示（頭尾雙指標法）：**
昨天你學的「快慢指標」是兩個指標從左邊一起出發；今天我們要用的是**「頭尾指標（對向移動）」**！
1. 宣告一個左指標 `left_ptr = 0`。
2. 宣告一個右指標 `right_ptr = arr_size - 1`（指向陣列最後一格）。
3. **核心邏輯：** 寫一個 `while` 迴圈，條件是只要 `left_ptr < right_ptr`（兩個人還沒相撞）：
* 呼叫你昨天寫得很完美的 `swap(s[left_ptr], s[right_ptr])`。
* 左指標往右走一步（`left_ptr++`）。
* 右指標往左走一步（`right_ptr--`）。



class Solution {
public:
    void reverseString(vector<char>& s) {
        int left_ptr=0;
        int right_ptr=s.size()-1;
        while(left_ptr < right_ptr){
            char temp='0';
            temp = s[left_ptr];
            s[left_ptr] = s[right_ptr];
            s[right_ptr] = temp;
            left_ptr++;
            right_ptr--;
        }
    }
};
