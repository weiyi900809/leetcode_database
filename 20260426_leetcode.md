160. Intersection of Two Linked Lists


/**
 * Definition for singly-linked list.
 * struct ListNode {
 * int val;
 * ListNode *next;
 * ListNode(int x) : val(x), next(NULL) {}
 * };
 */
class Solution {
public:
    ListNode *getIntersectionNode(ListNode *headA, ListNode *headB) {
        // ==========================================
        // 📍 浪漫雙指標起點：分別站在兩條路線的起跑線上
        // ==========================================
        ListNode* pA = headA; 
        ListNode* pB = headB;
        
        // 只要兩人還沒相遇 (指向同一個記憶體位址)，就繼續往下走
        // 💡 防呆機制：如果兩條 List 根本沒有交集，pA 和 pB 最後會同時變成 nullptr，
        // 迴圈條件 (nullptr != nullptr) 為 false，也會安全結束！
        while (pA != pB) {
            
            // 🏃‍♂️ A 的步伐：
            // 如果 A 已經走到路線盡頭 (nullptr)，下一步就瞬間跳轉到 B 路線的起點。
            // 否則，正常往前走一格。
            pA = (pA == nullptr) ? headB : pA->next;
            
            // 🏃‍♀️ B 的步伐：
            // 如果 B 已經走到路線盡頭 (nullptr)，下一步就瞬間跳轉到 A 路線的起點。
            // 否則，正常往前走一格。
            pB = (pB == nullptr) ? headA : pB->next;
        }
        
        // 迴圈結束時，他們要嘛同時站在「交會點」，要嘛同時站在「nullptr (代表無交集)」
        return pA;
    }
};



### 為什麼這個魔法會成立？(嚴謹的數學證明)



這看似神奇的「浪漫相遇」，背後其實是無懈可擊的數學規律。我們可以把這兩條 Linked List 拆解成三個部分來計算長度：

1.  設 **$a$** 為 List A 在交會點之前的長度。
2.  設 **$b$** 為 List B 在交會點之前的長度。
3.  設 **$c$** 為兩條 List 交會之後的「共同長度」。

也就是說：
* List A 的總長度是 $a + c$
* List B 的總長度是 $b + c$

**情況一：兩條路線有交集 ($c > 0$)**

當指標 `pA` 和 `pB` 開始走的時候，它們在第一趟絕對碰不到面（除非 $a = b$）。
但是，我們強制規定「走到盡頭就去走對方的路」，於是神奇的事情發生了：

* **指標 `pA` 抵達交會點所走過的總步數：**
    先走完 A 路線 ($a + c$) $\rightarrow$ 跳到 B 路線起點 $\rightarrow$ 走到交會點 ($b$)
    總步數 = **$a + c + b$**

* **指標 `pB` 抵達交會點所走過的總步數：**
    先走完 B 路線 ($b + c$) $\rightarrow$ 跳到 A 路線起點 $\rightarrow$ 走到交會點 ($a$)
    總步數 = **$b + c + a$**

因為在數學上 $a + c + b = b + c + a$，這證明了：**在第二趟的時候，兩個指標絕對會以「完全一樣的步數」，在交會節點上精準撞見彼此！**

**情況二：兩條路線完全沒有交集 ($c = 0$)**

如果它們從頭到尾都沒有相交，這個演算法會不會變成無限迴圈？
不會！因為套用一樣的數學公式：
* `pA` 走完 A 再走完 B，總步數是 $a + b$。
* `pB` 走完 B 再走完 A，總步數是 $b + a$。

當它們各自走完 $a + b$ 步時，`pA` 剛好走到 B 的盡頭變成了 `nullptr`，`pB` 剛好走到 A 的盡頭變成了 `nullptr`。這時候 `pA == pB` (`nullptr == nullptr`) 成立，迴圈優雅地結束，並回傳 `nullptr`，精準宣告這兩條路線沒有交集。
