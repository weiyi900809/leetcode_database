235. Lowest Common Ancestor of a Binary Search Tree

/**
 * Definition for a binary tree node.
 * struct TreeNode {
 * int val;
 * TreeNode *left;
 * TreeNode *right;
 * TreeNode(int x) : val(x), left(NULL), right(NULL) {}
 * };
 */

class Solution {
public:
    TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
        // 情境 A：兩個人都在左部門，總裁把任務往下交給左副總
        if (p->val < root->val && q->val < root->val) {
            return lowestCommonAncestor(root->left, p, q);
        }
        
        // 情境 B：兩個人都在右部門，總裁把任務往下交給右副總
        if (p->val > root->val && q->val > root->val) {
            return lowestCommonAncestor(root->right, p, q);
        }
        
        // 情境 C：消去法！
        // 既然沒有掉進上面兩個 if，代表他們沒有都在同一邊。
        // 這意味著他們在這裡「分道揚鑣」了 (一左一右)，或者總裁自己就是其中一人。
        // 這個分開他們的交叉點，就是我們要找的共同大主管！
        return root;
    }
};

你把前兩個情境的邏輯抓得非常精準！這正是 BST 最強大的地方：利用數值大小直接淘汰掉一半的樹。

你遇到的第三個問題：「`p` 和 `q` 可能一個大一個小，而且不知道誰大誰小，這要怎麼寫才乾淨？」

這是一個非常棒的邏輯盲點！這題最優雅的解法，其實是利用程式邏輯中的**「消去法 (Process of Elimination)」**。

### 💡 嚴謹邏輯破解：為什麼不用寫出第三種條件？

我們來盤點一下總裁看著這兩個數字，所有可能的排列組合：
1. 兩個人都比總裁小。
2. 兩個人都比總裁大。
3. `p` 比總裁小，`q` 比總裁大。
4. `p` 比總裁大，`q` 比總裁小。
5. `p` 就是總裁本人（`q` 在底下）。
6. `q` 就是總裁本人（`p` 在底下）。

如果我們在程式碼裡，已經用兩個 `if` 把 **第 1 種** (都在左) 和 **第 2 種** (都在右) 的狀況攔截下來，並叫他們繼續往下遞迴了。

那麼，如果程式能順利躲過這兩個 `if` 走到最後，**剩下的狀況 (第 3, 4, 5, 6 種)，全部都代表同一件事：這兩個人在這裡「分道揚鑣」了！**

一旦兩個人在目前這個總裁 (`root`) 的面前，一個要往左走、一個要往右走（或者其中一個人剛好走到自己了），那就代表**這個總裁就是把他們分開的「交叉點」**。
在樹的結構裡，這個「交叉點」，**絕對就是他們的最近共同祖先 (LCA)！**

###  業界標準的極致優雅寫法

所以，你根本不需要去寫落落長的 `(p->val < root->val && q->val > root->val)`。你只需要善用 `if` 的過濾特性，把剩下的狀況直接 `return root;` 結案：

```cpp
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 * int val;
 * TreeNode *left;
 * TreeNode *right;
 * TreeNode(int x) : val(x), left(NULL), right(NULL) {}
 * };
 */

class Solution {
public:
    TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
        // 情境 A：兩個人都在左部門，總裁把任務往下交給左副總
        if (p->val < root->val && q->val < root->val) {
            return lowestCommonAncestor(root->left, p, q);
        }
        
        // 情境 B：兩個人都在右部門，總裁把任務往下交給右副總
        if (p->val > root->val && q->val > root->val) {
            return lowestCommonAncestor(root->right, p, q);
        }
        
        // 情境 C：消去法！
        // 既然沒有掉進上面兩個 if，代表他們沒有都在同一邊。
        // 這意味著他們在這裡「分道揚鑣」了 (一左一右)，或者總裁自己就是其中一人。
        // 這個分開他們的交叉點，就是我們要找的共同大主管！
        return root;
    }
};
```
