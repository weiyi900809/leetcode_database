
20. Valid Parentheses (Easy) 

* **DV 實戰意義：** 在寫腳本解析 Verilog 或 C++ 原始碼的語法樹（Syntax Tree），或是檢查設定檔（Config Files）的格式是否合法時，括號的成對匹配是最基礎的防禦機制。
* **題目目標：** 給你一個只包含 `(`、`)`、`{`、`}`、`[`、`]` 的字串，請判斷這些括號是否合法閉合（必須以正確的順序、同類型的括號互相配對）。
* **破關提示（Stack 堆疊法）：**
這題是全 LeetCode 最經典的 **Stack（堆疊，後進先出 LIFO）** 應用題。這就像硬體裡的 LIFO Buffer。
1. 在 C++ 準備一個 `std::stack<char>`。
2. 用 `for` 迴圈把字串裡的字元一個一個讀出來。
3. **核心邏輯：**
* 遇到「左括號」 `(` 或 `{` 或 `[`：不要懷疑，直接把它**推入（Push）**堆疊裡等待配對。
* 遇到「右括號」 `)` 或 `}` 或 `]`：這時候就要檢查了！
* 防呆檢查：如果堆疊是空的，代表沒有左括號可以配對，直接 `return false`。
* 配對檢查：看看堆疊**最上層（Top）**的那個左括號，是不是跟現在這個右括號同一國的？如果是，就把最上層的左括號**彈出（Pop）**；如果不是，立刻 `return false`。


4. 迴圈跑完後，還有最後一道防線：檢查堆疊是不是「全空」。如果裡面還有剩下來的左括號，代表沒配對完，也要回傳 `false`。




class Solution {
public:
    bool isValid(string s) {
        stack<char> a;
        int arr_size = s.size();
        
        for(int i = 0; i < arr_size; i++){
            // 遇到左括號，直接 Push
            if(s[i] == '(' || s[i] == '{' || s[i] == '[' ){
                a.push(s[i]);
            }
            // 遇到右括號，開始檢查
            else if(s[i] == ')' || s[i] == '}' || s[i] == ']' ){
                
                // 【修正 1】防呆：如果堆疊是空的，代表右括號多出來了，絕對配對失敗
                if(a.empty()){
                    return false;
                }
                
                char temp = a.top();
                
                // 檢查是否不匹配
                if(s[i] == ')'  && temp != '('){
                    return false;
                }
                if( s[i] == '}' && temp != '{'){
                    return false;
                }
                if( s[i] == ']' && temp != '['){
                    return false; 
                }
                
                // 【修正 2】如果順利通過上面三個 if，代表「配對成功」
                // 必須把最上層的左括號彈出（抵銷掉）
                a.pop();
            }
            else{
                return false;
            }
        }
        
        // 迴圈跑完後，檢查堆疊是否清空。可以直接簡化寫法 
        return a.empty(); 
    }
};

14. Longest Common Prefix (Easy)




* **題目目標：** 給你一個包含許多字串的陣列（例如 `["flower","flow","flight"]`），請找出它們所有人共同擁有的「最長開頭字串」（答案是 `"fl"`）。如果沒有共同字首，回傳空字串 `""`。
* **破關提示（垂直掃描法）：**
這題考驗的是陣列與字串的雙重 Index 控制。
1. **防呆機制：** 如果輸入的陣列是空的，直接回傳 `""`。
2. **基準線：** 直接拿陣列裡的「第一個字串」當作對照組（基準）。
3. **雙層檢查邏輯：**
* 外層迴圈：走訪基準字串的每一個字元（Index `i`）。
* 內層迴圈：拿著這個字元，去跟陣列裡「其他所有的字串」的第 `i` 個字元做比對。


4. **終止條件（觸發回傳的瞬間）：** 只要發生以下兩種情況之一，就代表共同字首已經到極限了：
* 其他字串的長度不夠了（也就是 `i` 已經等於某個字串的總長度）。
* 其他字串在第 `i` 個位置的字元，跟基準字串不一樣。


5. 只要觸發終止條件，直接用 C++ 字串的 `substr` 函式，把基準字串從頭切到 `i` 回傳即可。

這是一個非常敏銳的問題！你注意到了 `racecar` 和 `car` 的相似之處，這也是這題（14. Longest Common Prefix）最常騙到人的文字陷阱。

關鍵在於對 **「Prefix（字首 / 前綴）」** 這個專有名詞的嚴格定義。

在電腦科學與字串處理中，**Prefix 必須「從 index 0（第一個字元）開始一模一樣」**，不能從中間或尾巴開始算。

我們用嚴謹的字元比對（Index by Index）來拆解這個例子：

* **字串 1 (`dog`)：** 第 0 格是 `d`
* **字串 2 (`racecar`)：** 第 0 格是 `r`
* **字串 3 (`car`)：** 第 0 格是 `c`

### 為什麼答案不是 "car"？

1. **位置不對（Prefix vs. Suffix）：** 雖然 `racecar` 裡面確實包含 `car`，但它是出現在字串的最後面。這在定義上叫做 **Suffix（字尾 / 後綴）** 或是 **Substring（子字串）**，絕對不能稱為 Prefix。
2. **必須「全部人」都有：** 題目的要求是找出陣列中「所有」字串的共同字首。這三個字串連最前面的第一個字元（`d`, `r`, `c`）都完全不一樣，在第一關就比對失敗了，所以它們的共同字首長度是 0，只能回傳空字串 `""`。

----

class Solution {
public:
    string longestCommonPrefix(vector<string>& strs) {
        int arr_size = strs.size();
        if(arr_size == 0) return "";
        
        string first = strs[0];
        int first_size = first.size();
        
        for(int i = 0; i < first_size; i++){
            for(int j = 1; j < arr_size; j++){
                string temp = strs[j];
                
                // 【嚴謹防呆修正】
                // 條件 1: i 已經撞到 temp 的總長度了 (代表 temp 比較短，提前結束)
                // 條件 2: 發現字元不一樣
                if(i == temp.size() || first[i] != temp[i]){
                    return first.substr(0, i); // i 剛好是截取字串的總長度
                }
            }
        }
        
        // 【補上 Missing Return】
        // 如果迴圈全部跑完都沒被 return 擋下來，代表 first 全員過關！
        return first; 
    }
};
