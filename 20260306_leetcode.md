leetcode 136 Single Number


class Solution {
public:
    int singleNumber(vector<int>& nums) {
        int result=0;
        int length=nums.size();
        for (int i=0;i<length;i++){
            result ^= nums[i];
        }
        return result;

    }
};

這是一個非常真實、而且幾乎所有剛開始面試的工程師都會經歷的「撞牆期」。你現在會覺得很慌，是因為你把面試解題當成了「發明創造」，覺得自己必須要能夠「靈光一閃」才能想到最佳解。

但業界的殘酷真相是：**這根本不是靈光一閃，這只是單純的「條件反射（Pattern Matching）」。**

你現在連不到那邊，不是因為你邏輯不好，單純只是因為你腦海裡的「面試關鍵字 Lookup Table (LUT)」還沒有建立起來。沒有人是天生就知道「成對出現 + 空間限制 = 位元運算」的。大家都是刷了題、背了這個 Pattern，下次看到才能瞬間反應。

我們用硬體設計最熟悉的邏輯來拆解，幫你把這個大腦裡的 LUT 建立起來。這是一套非常嚴謹且防呆的面試條件反射機制：

### 1. 建立大腦的「關鍵字 LUT (查找表)」

以後看到面試題，不要去想「怎麼解」，而是去抓「關鍵字」，讓關鍵字自動對應出你的武器庫。

* **觸發條件 A：看到「限制 $O(1)$ 空間複雜度 / 不能用額外記憶體」**
* **大腦強制輸出：** 絕對不能用 Hash Map、不能開新陣列。武器只剩下：**雙指標 (Two Pointers)** 或是 **位元運算 (Bitwise)**。


* **觸發條件 B：看到「成對出現 (Twice)」、「找唯一 (Single)」、「重複」**
* **大腦強制輸出：** 計算數量通常用 Hash Map，但如果有空間限制，武器只剩下：**XOR 抵銷** 或是 **先排序 (Sort) 再找**。


* **觸發條件 C：看到「已經排序好的陣列 (Sorted Array)」**
* **大腦強制輸出：** 武器高機率是 **二分搜尋法 (Binary Search)**。



當你有了這個 LUT，面試時你的大腦就會像跑電路邏輯一樣：
`Input: 限制空間 (A) + 成對出現 (B) -> 邏輯閘交集 -> Output: 嘗試使用 XOR。`
這需要靠我們前面列出的那 15 題來反覆訓練，把這個 LUT 燒進你的直覺裡。

### 2. 終極防呆機制：如果 LUT 真的當機了怎麼辦？

如果你連「排序」或「位元運算」都盤點不出來，腦袋真的完全當機，**請直接向面試官發出「中斷請求（Interrupt）」**，要提示。這絕對比你沉默五分鐘好上一百倍。

請把這句話當成求生咒語背下來：

> **「我目前只能想到用 Hash Map 掃描一次的暴力解，但我知道這不符合 $O(1)$ 的空間限制。要優化空間複雜度，我目前有點卡住，請問您可以給我一點關於『資料結構』或『演算法方向』的提示嗎？」**

這句話非常體面。你點出了暴力解、點出了限制，然後把球丟給面試官。面試官通常會直接說：「你有考慮過 Bit Manipulation 嗎？」這時候你的 LUT 就會被強制喚醒。
